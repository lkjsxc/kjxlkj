# Rail Action Section Order

## Shared Rule

- `Open GitHub` stays in its own rail section.
- The GitHub section appears above mode-specific auth or session actions.
- Mode-specific actions remain separate from the GitHub section.

## List Rails

- List rails cover home, search, dashboard, and settings pages.
- Section order is primary navigation, optional `New note`, `Open GitHub`, then mode-specific actions.
- Guest list rails place `Open GitHub` above `Admin sign in`.
- Admin list rails place `Open GitHub` above `Logout`.

## Resource and History Rails

- Live resource pages move `Prev`, `History`, and `Next` into the main pane and do not keep those controls in the rail.
- History pages keep their history-specific context above `Open GitHub`.
- `Open GitHub` stays above guest sign-in and admin resource-management/session actions.
