# Live Network Topology

## Edge Proxy

- `nginx` is the edge reverse proxy.
- It binds host TCP 80 and TCP 443.
- HTTP on 80 redirects to HTTPS on 443.
- HTTPS on 443 uses SNI-based stream routing with `ssl_preread`.

## SNI Routing

- Browser TLS ClientHello contains the SNI hostname.
- nginx `stream` block inspects SNI without decrypting.
- If SNI matches `turn.*` or `turns.*`, the raw TCP connection proxies to `coturn:5349` (TLS passthrough).
- For all other SNI values, the connection proxies to the internal nginx HTTPS listener on 8443.

## Service Map

```
Host 80   -> nginx:80   -> 301 redirect to 443
Host 443  -> nginx:443  -> stream SNI routing
  turn.*  -> coturn:5349 (TLS passthrough)
  default -> nginx:8443 (TLS termination) -> app:8080
```

## Internal Services

- `app`: Rust HTTP server on port 8080. Not exposed on the host.
- `coturn`: STUN/TURN on 3478/udp and 3478/tcp. TURN TLS on 5349/tcp internally.
- `postgres`: PostgreSQL on 5432 internally.
- `seaweedfs`: S3 gateway on 8333 internally.

## Certificates

- Local testing uses self-signed certificates generated by `scripts/generate-local-certs.sh`.
- The certificate SAN must include both the web domain and the TURN domain.
- Production operators mount real certificates into `nginx/ssl/` and `coturn/ssl/`.
- Let's Encrypt or other ACME providers are documented in [reverse-proxy.md](../../../operations/deployment/reverse-proxy.md).
