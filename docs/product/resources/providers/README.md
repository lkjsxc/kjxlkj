# Embed Providers

Use this subtree for provider-specific rules behind standalone external URL embeds.

## Read This Section When

- You are adding or changing a third-party provider.
- You need to know whether a provider renders as an iframe, hydrated social embed, or static card.
- You need fallback rules when provider URL parsing fails.

## Child Index

- [frame-providers.md](frame-providers.md): iframe and native-player provider rules
- [social-providers.md](social-providers.md): script-backed social embed rules
- [static-providers.md](static-providers.md): static rich-card provider rules
