# Cutover Checklist

## Pre-Cutover

- [ ] New tree exists with one `README.md` per directory.
- [ ] All stage and wave files are present.
- [ ] Checkpoint and evidence docs are present.

## Cutover

- [ ] Remove legacy restructuring paths.
- [ ] Update all affected TOCs and inbound links.
- [ ] Run structure and link audits.

## Post-Cutover

- [ ] Confirm no references to removed paths remain.
- [ ] Update final evidence status.
- [ ] Capture residual blockers in drift ledger.
