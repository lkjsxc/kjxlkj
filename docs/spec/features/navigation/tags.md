# Tags Support

Symbol-based navigation using tags files or LSP.

## Overview

Tags map symbol names to file locations, enabling "go to definition" and similar navigation. Tags can be generated by ctags or provided by LSP.

## Navigation Keybindings

| Key | Action |
|---|---|
| `<C-]>` | Jump to tag under cursor |
| `<C-t>` | Pop tag stack (go back) |
| `g]` | List all matching tags, select interactively |
| `g<C-]>` | Jump if one match, list if multiple |

## Tag Stack

Every `<C-]>` jump pushes the current position onto the tag stack. `<C-t>` pops back. `:tags` displays the stack.

The stack is per-window and unlimited depth.

## Commands

| Command | Action |
|---|---|
| `:tag {name}` | Jump to tag |
| `:tag` | Jump to last tag again |
| `:tnext` | Next matching tag |
| `:tprev` | Previous matching tag |
| `:tfirst` | First matching tag |
| `:tlast` | Last matching tag |
| `:tselect {name}` | List and select interactively |
| `:tags` | Show tag stack |
| `:pop` | Pop tag stack |

## Multiple Matches

When multiple tags match, `g]` or `:tselect` displays a numbered list showing file, line, and tag kind. Type a number to jump.

## Tag Kinds

| Kind | Description |
|---|---|
| `f` | Function / method |
| `c` | Class / struct |
| `m` | Module |
| `v` | Variable |
| `t` | Type / typedef |
| `e` | Enum variant |
| `i` | Interface / trait |

## Tags File Configuration

| Setting | Default | Description |
|---|---|---|
| `tags` | `"./tags;/"` | Semicolon-separated tag file search paths |
| `tagcase` | `"followic"` | Case sensitivity for tag matching |

The search path `./tags;/` means: look for `tags` in the file's directory, then upward to root.

## LSP Integration

When an LSP server is available, `<C-]>` prefers LSP `textDocument/definition` over tags files. Falls back to tags if LSP returns no result.
