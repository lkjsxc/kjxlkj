# Docs Layout Contract

## Mandatory Top-Level Directories

- `docs/getting-started/`
- `docs/vision/`
- `docs/product/`
- `docs/architecture/`
- `docs/operations/`
- `docs/repository/`

## Recursive Rule

Every docs directory has one `README.md` TOC plus multiple children.

## Navigation Rule

- `docs/getting-started/` is the canonical first-stop entry path.
- Deeper sections own stable contracts and runbooks after onboarding hands off.
- API contracts live under `docs/product/api/`.
