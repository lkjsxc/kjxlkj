# Docs Authoring Rules

## Writing Rules

- Prefer declarative requirements.
- Use stable headings and short bullet statements.
- Keep files under 300 lines.
- Start each directory `README.md` with purpose, reader intent, and a child index.
- Prefer task-first navigation before exhaustive taxonomy.
- Keep onboarding pages concise and link to deeper canon instead of duplicating it.

## Linking Rules

- Use relative links.
- Update parent TOCs when child entries change.
- Keep canonical definitions singular.
- When moving a rule, delete the old owner instead of leaving parallel copies.
